# 🧪 Exercise: Build a Todo App with GitHub Copilot

This is a hands-on exercise to practice using GitHub Copilot to generate a working web app from a single prompt.

---

## 🎯 What you'll build

A simple **Todo List** web app — entirely in one HTML file — with:

- An input field to add new tasks
- A list that displays all your todos
- A **Done** button to mark tasks as complete
- A **Delete** button to remove tasks

---

## 📋 Your task

1. Open a **new file** in VS Code (or any editor with Copilot enabled)
2. Make sure GitHub Copilot Chat is open (`Ctrl+Alt+I` / `Cmd+Option+I`)
3. Type the following prompt into Copilot Chat **exactly as written**:

---

> **Prompt to use:**
>
> *"Create a simple todo app with HTML, CSS, and JavaScript. Include an input field to add todos, a list to display them, and a delete button for each item."*

---

4. Review the code Copilot generates
5. Save the file as `my-todo-app.html`
6. Open it in your browser and test it works

---

## ✅ Success criteria

Your finished app should:

- [ ] Let you type a task and add it to the list
- [ ] Display all added tasks in a list
- [ ] Have a **Delete** button on each task that removes it
- [ ] Work entirely in the browser with no external dependencies

---

## 💡 Bonus challenges

Once you have the basics working, try asking Copilot to extend it:

- **"Add a Done button that strikes through completed tasks"**
- **"Save todos to localStorage so they persist on page refresh"**
- **"Add a filter to show All / Active / Completed tasks"**
- **"Style it to look like a dark mode app"**

Each of these is a single follow-up prompt — see how far you can get!

---

## 🔍 Things to notice

- How much working code did Copilot produce from one short prompt?
- Did it add anything you didn't ask for (e.g. animations, empty state messages)?
- How does changing the wording of your prompt change the output?
